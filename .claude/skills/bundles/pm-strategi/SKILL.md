# PM-Strategi Bundle

Samler alle strategiske PM-frameworks i ét bundle. Aktivér den skill der passer bedst til opgaven — de øvrige er tilgængelige som supplement.

## Skills i dette bundle

### Kerne-PM
| Skill | Aktivér når |
|---|---|
| `product-manager-skills:prd-development` | Skrive eller forfine et PRD (interaktivt interview-format) |
| `to-prd` | Konvertere eksisterende samtalekontext direkte til PRD — ingen interview |
| `product-manager-skills:roadmap-planning` | Planlægge eller kommunikere roadmap |
| `product-manager-skills:discovery-process` | Starte en discovery-fase |
| `product-manager-skills:prioritization-advisor` | Prioritere initiativer (RICE, MoSCoW) |
| `product-manager-skills:epic-breakdown-advisor` | Nedbryde epics til user stories |
| `product-manager-skills:user-story` | Skrive user stories |

### Discovery og brugerindsigt
| Skill | Aktivér når |
|---|---|
| `product-manager-skills:jobs-to-be-done` | Afdække JTBD og brugerens egentlige mål |
| `product-manager-skills:opportunity-solution-tree` | Kortlægge muligheder og løsningshypoteser |
| `product-manager-skills:discovery-interview-prep` | Forberede brugerinterviews |
| `product-manager-skills:problem-framing-canvas` | Formulere og indramme problemrummet |
| `product-manager-skills:problem-statement` | Skrive en skarp problemformulering |
| `product-manager-skills:proto-persona` | Definere primær brugerprofil |

### Strategi og positionering
| Skill | Aktivér når |
|---|---|
| `product-manager-skills:positioning-workshop` | Definere positionering og differentiering |
| `product-manager-skills:positioning-statement` | Skrive færdigt positioning statement |
| `product-manager-skills:product-strategy-session` | Overordnet produktstrategi |
| `product-manager-skills:tam-sam-som-calculator` | Beregne markedsstørrelse |
| `product-manager-skills:business-health-diagnostic` | Diagnosticere forretningens tilstand |
| `product-manager-skills:recommendation-canvas` | Formulere og forankre en anbefaling |

### Stakeholder og alignment
| Skill | Aktivér når |
|---|---|
| `product-manager-skills:pol-probe-advisor` | Kortlægge politiske dynamikker og stakeholders |
| `product-manager-skills:executive-onboarding-playbook` | Onboarde ny leder eller stakeholder |
| `stakeholder-map` | Power/Interest-grid med kommunikationsstrategi per kvadrant |
| `ubiquitous-language` | Etablere fælles domæneglossar — præcise termer der gælder på tværs af PM, tech og salg |
| `pre-mortem` | Risikoanalyse inden launch: Tigers, Paper Tigers og Elephants |
| `grill-with-docs` | Stress-teste strategi og beslutninger |
| `zoom-out` | Strategisk overblik inden beslutning |

### Output og refleksion
| Skill | Aktivér når |
|---|---|
| `outcome-analyse` | Evaluere afsluttet initiativ |
| `feedback-analyse` | Syntetisere brugerfeedback til PM-beslutning |
| `research-brief` | Specificere manglende ekstern viden |

## Brugsanvisning

1. Identificér den primære opgave og aktivér den mest præcise skill.
2. Brug `grill-with-docs` til systematisk stress-test af strategien.
3. Afslut med `research-brief` hvis ekstern viden mangler.
4. Brug `document-quality` (se præsentation-bundle) inden CEO-præsentation.

Aldrig to skills parallelt. Aktivér én, fuldfør, vurdér om en anden er nødvendig.
